As a language enthusiast, I am constantly looking for ways to practice my new language more. Everybody claims that the best way to learn a language is consistent practice speaking it. Grammar books and vocab lists certainly have their place, but from hands on experience learning spanish, I know that immersive practice was the key to unlocking eventual fluency.
Now, I have been working on learning hebrew for some time and find it much more difficult to find native speakers of the language with whom to practice. So, being a man of the 21st century, I decided to leverage the technologies at hand and practice with an LLM. While this was certainly better than nothing, it proved less helpful than I initially had hoped. The LLM would use vocabulary much beyond my scope and so I didn't even know where to start. I am sure that if I had more patience I eventually would have learned even better, but my patience in this case failed me. I contrasted that experience with my sessions practicing hebrew with a friend. We were both at similar levels, but had strengths in different domains and vocabulary subject matters. When one didn't understand a term, the other would circumlocute to help the other grasp the word on a deeper level. Had she instead circumlocuted using words I didn't know either (that being much my experience with the LLMs), I wouldn't have been able to build my vocabulary and comprehension on the foundation I had already laid.
Thus, my vision with this project is to build a chat bot that emulates these practice sessions. The system prompt needs to be thus that it knows it is supposed to pay attention to the vocabulary used by the user and try to match it, but push them just a little: circumlocuting words on which the user asks clarification in already familiar vocab. It can pull this from its context window, or perhaps from files/database lists of words specified by the user. The chatbot should attempt to always speak in hebrew, reducing unfamiliar terms to simpler and simpler hebrew, and as a last resort, showing a picture to the user of the concept it is trying to demonstrate.
The UI will appear like a normal chatbot, but when the user clicks on a word they don't know, the llm will generate definition (in hebrew) and place it in a popup window. There will also be a portion of the UI where the user can upload vocabulary lists, then checkboxes to tell the chatbot which ones he/she is already familiar with. It will include a dropdown at the top where the user can specify which model they want to converse with and a text box for them to put an API key. For now, it will be simple and only support one conversation, but in the future, there could be an option for more conversations.
This would be built using React for the front end, and for the prototype would store the files locally/in memory (they won't be large for now), but the functionality will be there to build out a backend that can manage user logins and file storage.
